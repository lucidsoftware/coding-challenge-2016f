Conveyor Loops

Summary

        Given a grid of conveyor belts, what is the longest loop an object placed on them could travel? Conveyor belts are 1m long and 1m wide, and move at the same speed (1m/s, why not).

Description

        Factorio is a game wherein the player must build an elaborate network of machinery and conveyor belts to produce increasingly complex goods until finally they build a rocket. New players quickly find that scalability is important, and that loops of conveyor belts can be useful.

        You are presented with a grid containing conveyor belts and empty spaces. Each conveyor belt is as wide as it is long (one meter squared) and can transport materials at a uniform speed. Each empty space is the same size (1m2). The speed and capacity of conveyor belts are irrelevant. Each conveyor belt may move objects UP, LEFT, DOWN, or RIGHT. Conveyor belts are static - they do not change direction. Empty spaces do not move resources.

        Any resource placed on the grid will eventually be stuck, or in a loop. An object is stuck if it ends up off the grid, or on an empty space. An object is in a loop if the path that it follows is infinitely repeating. The simplest example of this is two belts, the one on the left facing RIGHT and the one on the right facing LEFT. If the object starts on the LEFT belt, it travels one unit RIGHT, then one unit LEFT, RIGHT, LEFT, ad infinitum. The loop in this example has a size of two, since there are two belts on this infinitely repeating pattern.

The first line of input will be two space-separated integers: r and c, the number of rows and columns in the conveyor belt grid, respectively. The next r lines of input will all contain c space-separated characters: ^ (UP), > (RIGHT), < (LEFT), V (DOWN), or - (EMPTY). The output of your code will be a single integer k - the length of the largest loop of conveyor belts that is found on that grid. If no loops exist, this will be zero. The minimum length of a loop is 2.
